# 稳盈 / Poise · 数据源适配器接入指南

> 给客户 IT / 第三方实施方的「接入手册」——
> 实现一个 SourceAdapter 子类，把你方的 ERP / 银企 / Excel 数据接入稳盈的核心引擎。

---

## 1. 总体思路

稳盈核心引擎只认一套 **Canonical 数据契约**（7 张表的 dict 形态，字段定义见 [doc/数据契约_CSV.md](数据契约_CSV.md)）。

```
你方系统  →  [SourceAdapter.fetch()]  →  CanonicalDataset  →  importers  →  稳盈 DB  →  引擎
  ↑                                          ↑
  (异构: SAP / Oracle / 用友 / 金蝶 /        (统一: dict 形态)
   银企 / Excel / Kafka stream / ...)
```

**你方只需实现 `fetch()` 方法**，把异构数据映射成 Canonical dict 列表。其它（校验、入库、审计、容错）稳盈引擎全包。

---

## 2. 三种接入路径（从快到慢）

### 路径 A · CSV 上传（5 分钟）

最快路径：让客户/财务人员把已有 Excel 导出为 7 个 CSV 文件，按 [数据契约_CSV.md](数据契约_CSV.md) 字段名命名（`entities.csv` / `accounts.csv` / ...），在 `/data` 页点 「📤 CSV 上传」。

- ✅ 不写代码
- ✅ 数据质量门自动校验，错误行可见
- ⚠ 手工流程，不适合每日刷新

### 路径 B · Excel 工作簿（10 分钟）

一个 .xlsx 文件，7 个 sheet（同名）。调 `POST /api/v1/data/upload-excel` 上传一次即可。

- ✅ 不写代码
- ✅ 财务最熟悉的格式
- ⚠ 仍是手工流程

### 路径 C · 自定义 Adapter（1-3 天，生产推荐）

实现一个 SourceAdapter 子类，连接你方 ERP / 银企 API 自动拉取。

- ✅ 自动化，可定时拉取
- ✅ 一次实现，长期维护
- ✅ 复用稳盈所有校验/容错/审计

---

## 3. 写一个自定义 Adapter（路径 C）

### 3.1 模板

复制 `apps/api/poise/data_integration/adapters.py` 中的 `ExampleERPStub`：

```python
from poise.data_integration.adapters import (
    SourceAdapter,
    CanonicalDataset,
    register,
)


class ErpSapAdapter(SourceAdapter):
    name = "erp_sap"

    def __init__(self, *, connection_str: str, company_code: str = "DEMO"):
        self.connection_str = connection_str
        self.company_code = company_code

    def fetch(self, *, from_date: str | None = None, to_date: str | None = None,
              **kwargs) -> CanonicalDataset:
        # 1. 连接 SAP（pyrfc / sapnwrfc / requests 等）
        sap_data = self._query_sap(from_date, to_date)

        # 2. 映射到 Canonical
        ds = CanonicalDataset()
        ds.entities = [{"code": self.company_code, "name": "示范集团", "base_currency": "CNY"}]

        ds.accounts = [
            {
                "entity_code": self.company_code,
                "code": row["BUKRS"] + "-" + row["HKONT"],
                "name": row["TXT50"],
                "bank_name": row["BANKL"],
                "currency": row["WAERS"],
                "account_type": "basic",
            }
            for row in sap_data["accounts"]
        ]

        ds.cashflows = [
            {
                "entity_code": self.company_code,
                "account_code": row["HKONT"],
                "direction": "inflow" if row["BSCHL"] == "40" else "outflow",
                "category": _map_category(row["SAKNR"]),
                "source_type": "ar" if row["BSCHL"] == "40" else "ap",
                "expected_date": row["BUDAT"],
                "amount": str(row["DMBTR"]),
                "currency": row["WAERS"],
                "certainty_layer": _classify_layer(row["BUDAT"]),
                "counterparty": row.get("LIFNR") or row.get("KUNNR"),
            }
            for row in sap_data["postings"]
        ]

        # 同理 ds.balances / ds.instruments / ds.credit_lines / ds.reserve_rules

        return ds

register(ErpSapAdapter)
```

### 3.2 注册 + 调用

```bash
# 把 ErpSapAdapter 放到 apps/api/poise/data_integration/adapters.py 文件末尾
# 或新文件 apps/api/poise/data_integration/erp_sap.py（要保证被 import）

# 注册后立即可用
curl -X POST http://localhost:8001/api/v1/data/import-from-adapter \
  -H "Authorization: Bearer <token>" -H "Content-Type: application/json" \
  -d '{
    "adapter": "erp_sap",
    "kwargs": {
      "connection_str": "ASHOST=sap.example.com SYSNR=00 CLIENT=100",
      "company_code": "CN01",
      "from_date": "2026-06-01",
      "to_date": "2026-08-31"
    }
  }'
```

返回 `ImportSummary`，含每张表新增 / 跳过条数 + 全量 issue 列表。所有写入都会进 `AuditLog`，actor_role 标记为触发用户。

### 3.3 字段 mapping 速查（系统设计 §6.1）

| 你方字段 | 稳盈 Canonical |
|---|---|
| 法人主体编号 | `entity.code` |
| 银行账户 | `account.code` + `account.bank_name` |
| 账期末余额 | `balance.balance` / `available_balance` |
| 销售订单 / AR 票据 | `cashflow.direction=inflow, source_type=ar` |
| 采购订单 / AP 票据 | `cashflow.direction=outflow, source_type=ap` |
| 薪酬日程 | `cashflow.category=payroll, source_type=schedule` |
| 增值税日程 | `cashflow.category=tax, source_type=schedule` |
| 投资品种 + 利率 | `instrument.kind=invest, rate, tenor_options` |
| 授信额度 | `credit_line.limit_amount, used_amount, rate` |

## 4. 进阶：增量 upsert + 定时拉取

### 增量上传

`merge_canonical()` 工具函数按自然键 upsert，避免重复入库：

```python
from poise.data_integration.adapters import merge_canonical, get_adapter

adapter = get_adapter("erp_sap")
yesterday = adapter.fetch(from_date="2026-08-29", to_date="2026-08-29")
today = adapter.fetch(from_date="2026-08-30", to_date="2026-08-30")
merged = merge_canonical(yesterday, today)  # 自动按自然键去重
```

### 定时拉取

加 Celery Beat 任务（参考 Phase 6 的 `weekly_rolling_rerun`）：

```python
# apps/api/poise/data_integration/tasks.py
from poise.core.celery_app import celery_app
from poise.core.database import SessionLocal
from poise.data_integration.importers import import_from_adapter


@celery_app.task(name="poise.data_integration.tasks.daily_sap_pull")
def daily_sap_pull():
    with SessionLocal() as db:
        from datetime import date, timedelta
        yesterday = (date.today() - timedelta(days=1)).isoformat()
        import_from_adapter(
            db, "erp_sap",
            {"from_date": yesterday, "to_date": yesterday},
            actor_user_id=None, actor_role="scheduler",
        )
```

```python
# celery_app.py
celery_app.conf.beat_schedule["daily-sap-pull"] = {
    "task": "poise.data_integration.tasks.daily_sap_pull",
    "schedule": crontab(minute=0, hour=2),  # 每天 02:00
}
```

## 5. 安全建议

| 关注点 | 措施 |
|---|---|
| 连接串 / 密码 | 走 `.env` 注入，**不要**写在 adapter 代码里 |
| 审计 | 所有 import 自动落 `AuditLog`；连接串字段被自动脱敏 |
| 限流 | 在 Celery 队列层加 `rate_limit` 防雪崩 |
| 回滚 | 出错的事务自动 `db.rollback()`；预览模式 `commit=false` 先校验 |
| 数据脱敏 | 账户号、对手方名按需在 adapter 中做 hash |
