# 稳盈 / Poise · 用户手册

> 给企业财资人员（出纳、资金主管、财务经理）的使用指南。

---

## 1. 你是谁，能拿到什么

| 你的角色 | 能做什么 |
|---|---|
| **admin** 管理员 | 用户增删 / 角色分配 / 审计日志 + 全部业务能力 |
| **treasurer** 资金主管 | **采纳/否决方案** + 改假设 + 全部查看 + 审计日志 |
| **analyst** 出纳分析师 | 跑预测 / 求解 / 对话追问 / 改假设；**不能采纳方案** |
| **viewer** 只读 | 仅查看，不能改任何东西 |

登录：`http://localhost:3000/login`，开发期默认账号 `admin / treasurer / analyst / viewer`，密码 `Poise@2026`。

---

## 2. 一次完整的工作流程

### Step 1 · 导入数据（数据录入页）

第一次使用时点「**一键导入 demo_company 种子数据**」，会自动写入一份"稳盈示范实业"的 13 周完整剧情：
- 3 个银行账户 / 期初余额 ¥1.50 亿
- 39 条 13 周现金流（含 W3 一笔 ¥80M 并购款埋点）
- 10 个投资品种 + 3 个融资工具
- 3 条授信（流贷 80M / 票据 30M / 银承 20M）
- 备付规则：rolling_coverage = 4 周

**演示之后**：把上面的种子替换为你公司的真实 CSV（按相同字段，参见 `数据契约_CSV.md`）。

### Step 2 · 看预测（13 周看板页）

点「**生成首份预测**」或「重新预测」按钮。生成后页面显示：

- **顶部 4 卡**：期末余额（中性 vs 悲观）、硬性缺口周、擦边周
- **图 1 净现金流双情景**：橙色阴影是 uncertain 层的 ±20% 区间
- **图 2 余额 vs 备付**：红虚线是最低备付，蓝/橙线是中性/悲观余额；红橙 markPoint 标出缺口/擦边周
- **13 周明细表**：每周中性净 CF / 悲观净 CF / 备付 / 余额 / 安全垫 + 状态标签

**判读要点**：
- 安全垫 < 0 → **硬缺口**（红色）
- 安全垫 < 25% × 备付 → **擦边**（橙色）
- demo 数据下，悲观情景 W11 会触发擦边

### Step 3 · 求三档方案（方案对比页）

点「**重新求解（三档 MILP）**」。约 100ms 内出三套方案：

| 档位 | 风格 |
|---|---|
| 稳健 | 久期 ≤ 4 周，安全垫加 20%；保守 |
| 折中 | 久期 ≤ 12 周，安全垫不加成；推荐默认 |
| 进取 | 久期 ≤ 26 周，安全垫减 15%；贴近底线榨收益 |

**每张卡片显示**：
- 预期净收益（horizon 内）
- 投资动作笔数 / 融资动作笔数
- 融资依赖度（> 50% 触发"高依赖"标签）
- 缺口预警 / 高融资依赖 / 合规 三种状态 Badge
- 求解器状态（optimal 即可用）

下方两张曲线对比：安全垫曲线 + 演化余额曲线。

最下面是每档完整 PlanAction 明细（每周做什么 + 金额 + 期限 + 标的）。

**采纳/否决**（treasurer 角色才行）：在卡片底部点「采纳」按钮。系统会自动把同一份预测下的另两档标为 rejected，方案的 status 会变成 `adopted`（卡片高亮绿框）。

### Step 4 · what-if 沙盘

方案对比页右上角「**🧪 what-if 沙盘**」打开右抽屉：

- **资金锁定**：在某些周次"钉"住一笔资金（不可投、不可被授信替代）
- **快捷场景**：
  - 「并购款剧情（W3 锁 ¥80M）」—— 模拟必付的大额支出
  - 「月中工资周（W11 锁 ¥30M）」—— 模拟刚性人力成本
- 改完点「**应用锁定 + 重算三档**」，约 100ms 内出新方案

锁定过严会导致不可行 → 自动走 slack 诊断 → 缺口预警标签弹出。

### Step 5 · 对话参谋

最自然的入口。打开 `/chat`，直接问：

| 典型问题 | 期望流程 |
|---|---|
| 「未来 13 周哪几周会紧？」 | LLM 调 `run_forecast` → 摘要给你 |
| 「这周有什么闲钱能投？」 | 调 `build_and_solve` → 复述折中档首批动作 |
| 「下个月底的并购款付得出来吗？不够怎么补最省？」 | 调 `query_position` + `diagnose_infeasible` → 给出缺口规模与建议 |
| 「为什么折中比稳健收益高？」 | 调 `explain_plan` → 用业务语言讲清推理链 |
| 「假设北辰电气回款延后 2 周？」 | 调 `apply_overrides` → 解析假设 → dry-run 预览 |
| 「悲观情景下有哪些缺口风险？」 | 调多个 tool → 综合预警 |

**数字纪律（重要）**：LLM 绝不会凭空生成金额。所有数字都来自后端 tool 的真实返回。如果某次回复让你觉得"哪里不对"，展开对应消息下的「工具返回」JSON 自检。

### Step 6 · 预警中心

`/alerts` 聚合两个来源的风险：
- **预测引擎**：悲观情景的硬缺口周 + 擦边周
- **决策引擎**：求解后仍有缺口的方案 + 高融资依赖方案

每周自动滚动后这里会刷新。

### Step 7 · MAPE 看板 + 反馈学习

`/accuracy` 页面：
- 每周一 06:00 自动滚动（Celery Beat），也可手动「**手动触发滚动重跑**」
- 分层 / 分类 MAPE 柱状图：绿色 < 5%，蓝色 < 15%，橙色 < 30%，红色 ≥ 30%
- 偏差校正系数表：(category, direction) 维度学到的乘子，自动应用到下一轮预测
- 滚动重跑历史：scheduler 标识为自动触发，user_id 标识为手动

## 3. 数字纪律：参谋的底线

稳盈是**参谋**，不是操盘手：

1. **不替你动钱**。建议必须由你（treasurer）在前端「采纳」才生效；采纳本身也只是流程动作，不会调银行接口。
2. **金额必定可溯源**。LLM 的每个金额都对应某次 tool 调用的真实返回；tool 的金额来自 DB 中实际写入的 forecast/plan 数据。
3. **报忧也是本分**。MILP 不可行不会被装作可行；缺口、擦边、高融资依赖都会直接告诉你。

## 4. 常见操作小贴士

- **登录失败**：检查 `.env` 中 `Poise@2026` 没改。生产环境务必让 admin 在管理后台改密码。
- **对话总等待**：DeepSeek V4 推理模式单轮 7-30s 正常；屏幕显示「正在思考...」。
- **求解超过 10s**：可能是 max_amount big-M 退化，看 `apps/api/poise/optimization/model.py` 的 `abs_cap` 是否被改过。
- **导入失败**：CSV 字段不符规范时按 `quality_gate.py` 的 issue 列表逐行报错；error 级别阻止入库。

## 5. 我能在哪里看到所有操作记录

`/admin` → 审计日志（admin / treasurer 角色可访问）

支持按事件类型 / 角色 / 时间维度过滤。所有写操作 + LLM 调用 + 方案采纳/否决 + 滚动重跑都会留痕。
