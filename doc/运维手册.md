# 稳盈 / Poise · 运维手册

> 部署、监控、排错、应急。

## 1. 系统拓扑

```
                              ┌──────────────┐
                Browser ────► │  Next.js web │ (port 3000)
                              └──────┬───────┘
                                     │ NextAuth JWT
                                     ▼
                              ┌──────────────┐    ┌────────────┐
                              │ FastAPI api  │───►│ PostgreSQL │
                              │  (port 8001) │    │ (port 5632)│
                              └──┬───────┬───┘    └────────────┘
                                 │       │
                                 │       └──────────────────────────┐
                                 ▼                                  ▼
                          ┌────────────┐                    ┌──────────────┐
                          │   Redis    │◄───────────────────┤ celery-worker│
                          │ (port 6679)│                    └──────────────┘
                          └─────┬──────┘                    ┌──────────────┐
                                └────────────────────────── │  celery-beat │
                                                            └──────────────┘
```

6 个容器：`postgres / redis / api / web / celery-worker / celery-beat`。
共享镜像：`poise-cfls-api:dev` 同时被 api / celery-worker / celery-beat 使用。

## 2. 启动 / 停止

```bash
make up      # 启动全栈（docker compose up -d --build）
make down    # 停止
make logs    # tail api 日志
docker compose ps                  # 查看健康
docker compose logs <service> -f   # 跟某个服务
```

首次启动后：

```bash
docker compose exec api alembic upgrade head      # 应用迁移
docker compose exec api python -m poise.scripts.bootstrap   # 创建默认 4 个用户
make seed                                          # 导入 demo 种子数据
```

## 3. 配置 `.env`

复制 `.env.example` 到 `.env`，关键项：

```bash
# DB
DATABASE_URL=postgresql+psycopg://poise:poise_dev@postgres:5432/poise

# LLM（DeepSeek 默认）
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-...                  # 必须填，否则 chat 路径 503
DEEPSEEK_MODEL_CHAT=deepseek-chat        # V4 thinking 模型
DEEPSEEK_MODEL_LITE=deepseek-chat        # 或更便宜的 lite

# 求解器
SOLVER_BACKEND=cbc                        # 默认 cbc；pulp+highspy 兼容修复后可切 highs
SOLVER_TIME_LIMIT_SEC=10

# 鉴权
API_SECRET_KEY=<openssl rand -hex 32>     # 生产环境必改
NEXTAUTH_SECRET=<openssl rand -hex 32>
```

**修改 `.env` 后必须** `docker compose up -d`（而非 restart）才能让容器读到新值。

## 4. 数据库迁移

```bash
make migrate                                       # 应用所有未应用的迁移
make revision m='add column foo to bar'            # 生成新迁移（自动 diff schema）
docker compose exec api alembic downgrade -1       # 回滚一步
```

Alembic 版本表 `alembic_version` 在 `poise` DB。

## 5. 日志与监控

| 数据源 | 怎么看 |
|---|---|
| API 业务日志 | `docker compose logs api -f`（structlog JSON 行格式） |
| Celery 任务 | `docker compose logs celery-worker -f` |
| Celery Beat 调度 | `docker compose logs celery-beat -f` |
| **审计事件** | `/admin` 页面 → 审计日志 tab（按时间/角色/事件类型过滤） |
| 数据库 | `docker compose exec postgres psql -U poise -d poise` |

关键审计事件：

| event_type | 含义 |
|---|---|
| `http.request` | 所有 POST/PUT/PATCH/DELETE 写操作 |
| `data.import_demo` | 种子数据导入 |
| `forecast.run` | 触发预测 |
| `plan.build_and_solve` | 触发三档求解 |
| `plan.adopt` / `plan.reject` | 方案采纳/否决 |
| `llm.chat_turn` | LLM 对话（含 token 计数 + tool 序列） |
| `feedback.rolling_run` | 滚动重跑（scheduler / 手动） |
| `admin.user_create` / `admin.user_update` | 用户管理 |

## 6. 性能基线（demo 数据规模）

| 操作 | 实测耗时 |
|---|---|
| 触发预测 13 周双情景 | < 20 ms |
| 三档 MILP 求解（CBC） | 80-100 ms |
| what-if 重算 | 60-80 ms |
| 滚动重跑（不重新预测） | < 10 ms |
| LLM 单轮对话（DeepSeek V4 thinking） | 7-30 s |
| 冒烟全流程（不含 LLM） | < 2 s |

超出 2-3 倍属正常波动，超出 10 倍排查。

## 7. 应急 / 排错

### API 起不来
```bash
docker compose logs api --tail 50
```
常见原因：
- 端口冲突 → 改 `docker-compose.yml` 中的宿主映射（默认已避开常见冲突）
- Alembic 迁移失败 → 看堆栈，必要时 `alembic downgrade base && alembic upgrade head`
- pulp 缺失 → 镜像未重建 `docker compose build api`

### Chat 路径返回 503
- 检查 `.env` 的 `DEEPSEEK_API_KEY` 是否真实有效
- `docker compose up -d api` 让新 env 生效
- DeepSeek 平台余额是否充足

### MILP 不可行（plans `infeasibility` 非 null）
- 走 `/api/v1/feedback/...` 或 `/chat` 让 LLM 调 `diagnose_infeasible`
- 多半是 ReserveRule 过严或资金锁定过多 → 在 what-if 抽屉清空锁定看是否恢复

### Celery 任务不跑
```bash
docker compose logs celery-worker | grep tasks   # 应看到 weekly_rolling_rerun
docker compose logs celery-beat --tail 30        # 看 Beat 是否在 tick
```
若 worker 没注册任务 → 镜像未重建后 `include=["poise.feedback.tasks"]` 没被加载。

### 数据脏了想清重来
```bash
docker compose down -v        # 删数据卷（⚠ 会清掉所有数据）
docker compose up -d
docker compose exec api alembic upgrade head
docker compose exec api python -m poise.scripts.bootstrap
make seed
```

## 8. 备份

生产环境建议每日 PG dump：
```bash
docker compose exec postgres pg_dump -U poise poise > backup-$(date +%F).sql
```

恢复：
```bash
docker compose exec -T postgres psql -U poise -d poise < backup-2026-05-30.sql
```

## 9. 升级路径（集团演进）

系统设计 §8 留好的钩子：
- 所有实体带 `entity_id`；从单值扩为 list，配合 `entity_filter` 即可
- MILP `build_model(entities=[...])` 参数已留；要加内部调剂动作（上存下借 / 委贷）
- 视图层从单主体改为合并视图聚合

不需要重写引擎结构。

## 10. 安全清单（试点前必看）
