# demo_company · 稳盈 13 周完整剧情种子数据

## 企业画像

- **名称**：稳盈示范实业有限公司（DEMO）
- **行业**：制造 + 贸易混合
- **年营收**：~10 亿（月度回款节奏明显）
- **银行账户**：3 个（基本户 / 一般户 / 收款专户）
- **本币**：CNY

## 剧情埋点（对齐需求文档 US-1~US-6）

| 周次 | 事件 | 呼应用户故事 |
|---|---|---|
| W3 | 大额并购付款 **¥80,000,000**（一次性 outflow） | US-3 缺口场景 |
| W6 | 重点客户 *北辰电气* 回款 ¥35M 延后 2 周到 W8 | US-5 what-if 假设修改 |
| W9–W13 | 销售回款进入 *uncertain* 层，输出区间 | US-1 远端不确定区间预测 |
| W11 | 悲观情景下安全垫转负，触发缺口预警 | US-6 主动告警 |

## 期初状态

- **as-of date**：当前业务日 D0 = 2026-05-30（W0）
- **期初总余额**：¥150,000,000（基本户 100M / 一般户 30M / 专户 20M）
- **最低备付规则**：rolling_coverage，4 周（覆盖未来 4 周刚性支出）
- **risk_knob 默认**：折中

## 预期 demo 路径

1. 登录 → 数据录入 → "导入 demo_company"
2. 13 周看板：看到中性情景下 W3 安全垫陡降；悲观情景下 W11 安全垫转负
3. 方案对比：3 套 risk_knob（稳健/折中/进取），都把闲钱配到不同久期组合
4. 对话："下周的并购款付得出来吗？不够的话怎么补最省？" → 触发 build_and_solve + diagnose_infeasible
5. what-if："假设北辰电气回款再延 2 周" → 重算
6. 预警中心：W11 缺口预警 + 高融资依赖标签
